Core Technologies

React

UI Framework

React is a JavaScript library that lets you build user interfaces out of reusable pieces called components. Instead of manually updating the HTML every time data changes, React watches your data and automatically re-renders only the parts of the page that need to update.

Why it's used here

The brief generator has several pieces of state that need to stay in sync — the form inputs, the loading status, the output text streaming in word by word, whether the copy button has been clicked. React manages all of this automatically. Without it, you'd be writing a lot of messy manual DOM updates.

Content ops analogy: React is like a CMS template system. You define the structure once (the component), and it fills in the right content automatically based on what data it receives — just like a CMS renders the same article template with different content for every post.

useState & useCallback

React Hooks

useState is how React components remember things. Every time the user types in the Topic field, useState stores that value and makes it available to the rest of the app. useCallback optimizes functions so React doesn't recreate them unnecessarily on every render.

Why it's used here

The app tracks: all 7 form fields, the current generation status (idle/generating/done/error), the streaming output text, the error message, and whether the copy button was clicked. Each one is a piece of state. useState is the mechanism that holds all of it.

Content ops analogy: useState is like a live Google Doc that everyone on the team can see update in real time. When you change a field, every part of the app that depends on that field re-renders with the new value instantly.

Fetch API & Async/Await

JavaScript / HTTP

The Fetch API is the browser's built-in tool for making HTTP requests — sending data to and receiving data from external servers. async/await is syntax that makes it readable: it lets you write code that waits for a response before proceeding, without creating callback hell.

Why it's used here

When the user hits Generate, the app sends a POST request to the Anthropic API with the form data as the prompt. Fetch handles the request, async/await makes sure the app waits for the response before trying to process it, and the try/catch block handles errors gracefully if the API fails.

Content ops analogy: Fetch is like a courier between your app and the AI. You hand it a package (the prompt), it delivers it to Anthropic's servers, waits for a response package, and brings it back. async/await just means "wait at the door until the courier returns."

Anthropic API + Streaming

AI / Server-Sent Events

The Anthropic API is the interface that lets applications send prompts to Claude and receive AI-generated responses. Streaming (via Server-Sent Events) means the response comes back word-by-word in real time rather than all at once after a long wait — creating the typewriter effect you see in the output panel.

Why it's used here

A full brief can take 10–15 seconds to generate. Without streaming, the user stares at a spinner for the entire duration. With streaming, they see text appearing immediately — which feels responsive and fast even though the total time is the same. The app reads each chunk as it arrives, appends it to the output string, and React re-renders the display.

Content ops analogy: Non-streaming is like waiting for an entire article to be written before you can read any of it. Streaming is like watching someone write it in a live Google Doc — you can read along as it forms, which feels much more immediate.

Prompt Engineering

AI Input Design

Prompt engineering is the practice of crafting the instructions you send to an AI model to reliably produce the output you want. It involves specifying role, context, format requirements, and constraints in a structured way.

Why it's used here

The buildPrompt() function in the app takes the user's inputs and assembles them into a carefully structured prompt that tells Claude exactly what role to play, what inputs it has, and what structure the output must follow. This is where your content strategy expertise is genuinely irreplaceable — you know what a good brief contains better than any engineer who just learned what a brief is.

Content ops analogy: Prompt engineering is like writing a creative brief for the AI. The better and more specific your brief to the AI, the better the output it produces — exactly the same principle as briefing a human writer.

CSS Custom Properties

Styling / Design System

CSS custom properties (also called CSS variables) let you define design values — colors, spacing, fonts — once in a central place using :root { --green: #00E5A0 } and then reference them throughout your stylesheet. Changing the value in one place updates it everywhere.

Why it's used here

The entire color palette, spacing system, and typography of BriefAI is defined in 15 CSS variables at the top of the stylesheet. This means the whole design is themeable from a single block of code — swap those values and the entire app changes appearance. It's also the foundation for building a proper design system as the app grows.

Content ops analogy: CSS variables are like a brand style guide that's actually enforced programmatically. Instead of hoping writers remember the brand's hex codes, the system references the source of truth directly every time.

Blob API & File Download

Browser API

The Blob API lets you create file-like objects in memory from strings or data. Combined with a dynamically created <a> tag and URL.createObjectURL(), it lets the browser trigger a file download without any server involvement.

Why it's used here

The Download .md button takes the brief text, wraps it in a Blob with text/markdown MIME type, creates a temporary download URL, and programmatically clicks an anchor tag — all in about 5 lines of code. The file downloads as a .md file named after the brief topic. No server, no upload, no backend needed.

Content ops analogy: This is the equivalent of hitting "Export" in a CMS — the app packages up the content and hands it to you as a file without needing to touch a server.

dangerouslySetInnerHTML + Custom Markdown Parser

React / Text Rendering

React normally escapes all HTML to prevent security issues. dangerouslySetInnerHTML is React's escape hatch that lets you inject raw HTML into the DOM. The custom renderMarkdown() function converts the AI's plain text output (with # headings, ** bold, - lists) into formatted HTML before injecting it.

Why it's used here

Claude returns the brief as markdown text. Without parsing, the user would see raw ## SEO Strategy instead of a styled heading. The parser converts headings, bold text, and bullet lists into real HTML elements so the output looks like a properly formatted document rather than raw text.

Content ops analogy: This is the rendering engine — the equivalent of a CMS taking raw markdown content and rendering it as styled HTML on a webpage. Same concept, implemented in about 20 lines of JavaScript.
How It All Flows Together
1

User fills the form

React · useState

Each keystroke calls updateField() which calls setForm() — a useState setter. React re-renders only the affected input and the character counter. The Generate button watches whether the three required fields are filled and enables itself automatically.

2

User hits Generate

JavaScript · buildPrompt()

The generate() function fires. It sets status to 'generating' (React re-renders the button to show a spinner), clears previous output, then calls buildPrompt(form) — which assembles the form values into a structured prompt string ready to send to the API.

3

Request sent to Anthropic API

Fetch API · async/await · stream: true

Fetch sends a POST request to api.anthropic.com/v1/messages with the prompt, model name, max tokens, and stream: true. The API validates the request and begins generating a response, sending it back as a stream of Server-Sent Events rather than waiting to send everything at once.

4

Streaming response processed chunk by chunk

ReadableStream · TextDecoder · JSON.parse

The app reads the response body as a stream using response.body.getReader(). Each chunk arrives as raw bytes, gets decoded to a string, split into lines, parsed as JSON, and if it contains text delta content, appended to the accumulated string. setOutput(accumulated) triggers a React re-render on every chunk — creating the live typewriter effect.

5

Output rendered as formatted HTML

renderMarkdown() · dangerouslySetInnerHTML

On every render, the current output string is passed through renderMarkdown() which converts markdown syntax to HTML. That HTML is injected into the output panel via dangerouslySetInnerHTML. During generation, a blinking cursor element is appended to the end so the user can see exactly where the text is up to.

6

User copies or downloads the brief

Clipboard API · Blob API

Copy uses navigator.clipboard.writeText(output) — one line. Download creates a Blob from the output string, generates a temporary object URL, creates an invisible anchor tag, sets the filename dynamically from the topic field, and programmatically clicks it. The URL is then revoked to free memory. Status reverts to 'idle' when the user clears.
